怎么给变量，常量取名

这涉及到编码规范，统一的规范便于团队合作，便于阅读其他人的代码。

一份简单的规范：

  1. 包名：全部是小写字母，然后由 . 分隔。包名作为命名空间，具有唯一性，推荐使用公司或组织域名的倒置，比如 com.apple.quicktime.v2 。但Java核心库不采用这种方式。
  2. 类名、接口名：帕斯卡命名法， SplitViewController
  3. 文件名：帕斯卡命名法 BlockOperation.java
  4. 变量名：驼峰命名法 studentNumber
  5. 方法名：驼峰命名法
  6. 常量名：大写字母，可用下划线分开 year WEEK_OF_MONTH

二者之间有细微的区别 

public class HelloWorld {
    public static void main(String[] args) {
        String s1 = "str";
        String s2 = "str";
        String s3 = new String("str");
        String s4 = new String("str");

        System.out.println(s3 == s4);  //false
        System.out.println(s1 == s2);  //true

        System.out.println(s3 == s1); //false
        System.out.println(s2 == s3); //false
    }
}

== 运算符号比较的是两个对象的引用是否一致，所以 s1, s2 引用了同一格对象，s3, s4 引用不同对象。出现不同是因为java使用了 字符串池(String Pool) 。

=String s1 = "str"= 在字符串池创建 "str" 并被s1引用，创建s2时再次引用 "str"。但 new 方法创建的对象是不同的。

String字符串不可变，那要合并字符串呢?

虽然不可变，但可以拼接，因为可以生成一个新的字符串对象，使用以下方式进行拼接：

  1. + 不仅可以连接 String 类型，还可以连接其他类型，+ 号的左右操作数调用期 toString 方法。 
    public class HelloWorld {
    public static void main(String[] args) {
        String s1 = "str";
        String s2 = "str";
        String s3 = s1 + s2;
        System.out.println(s3);  //strstr

        int n1 = 1;
        String s4 = s1 + n1;
        System.out.println(s4); //str1
    }
}
  2. concat 
    public class HelloWorld {
    public static void main(String[] args) {
        String s1 = "str";
        String s2 = s1.concat("str2");
        System.out.println(s2);  //strstr2
    }
}

可变字符串哪里特殊?

追加、删除、修改、插入和拼接等操作不会产生新的对象，称为 字符串缓冲区

StringBuffer StringBuilder,为什么有2个可变对象

StringBuffer 是线程安全的，方法支持线程同步，在单线程环境下会影响效率会影响效率；StringBuilder 是单线程版本。二者拥有相同的 api 。 

public class HelloWorld {
    public static void main(String[] args) {
        StringBuffer s1 = new StringBuffer();
        System.out.println(s1.length());   // 0
        System.out.println(s1.capacity()); // 缓冲区长度为16
        s1.append("StringBuffer");
        System.out.println(s1);            // StringBuffer
        System.out.println(s1.capacity()); // 缓冲区长度为16
        s1.append(s1);
        System.out.println(s1);            // StringBufferStringBuffer
        System.out.println(s1.capacity()); // 缓冲区长度为34
    }
}
