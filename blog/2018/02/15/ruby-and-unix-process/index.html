Ruby Unix 进程实例

这篇博文是理解 Unix 进程的笔记，如果你不想阅读二手文章，请看原版书。看完这篇文章或者《理解 Unix 进程》并不能让您深入理解进程，您还是需要阅读操作系统相关书籍。

前置知识

用户空间(user space), 内核与系统调用(system call)

现代操作系统划分了 内核空间(kernel space) 和 用户空间(user space), 计算机使用者通过 Unix 内核与计算机硬件进行交互（读写文件、发送数据、分配内存、播放媒体文件等等），依赖内核的程序比如文件系统、驱动程序等等。鉴于内核强大的能力，程序不可以直接访问内核，所有与内核的交互都通过 系统调用（System call） 完成。不借助 System call 你仍可以写程序，比如科学计算、操作字符串、算法题等等，但有些情况你必须调用内核来完成，比如像 nginx mysql之类永远运行的后台程序。

进程

进程: Unix 基石

进程是 Unix 的核心，代码都是在进程中执行的。比如当你运行一个ruby程序时， ruby example.rb 你的代码就生成了一个新的进程，程序执行完毕，进程也随之退出。同样，运行 MySQL、PG 等数据库时，打开浏览器时，这些程序都运行在自己的进程中。

进程的标识符 pid

每个进程都有唯一的进程标识符（pid），对内核来说只是一个数字。我们可以编写一个 ruby 脚本查看其pid。 

puts Process.pid
puts $$

$$ 等同于 Process.pid ，这种用法来源于 perl 和 bash

我们还可以通过 ps 命令查看进程 

ps -p firefox<tab>

实际应用过程中，我们常常会在日志文件中发现pid。比如 apache 服务器的错误日志 /var/log/apache2/error.log 

[Wed Jan 31 13:36:52.665744 2018] [core:error] [pid 13639:tid 140581988599552] [client ::1:4866] AH00037: Symbolic link not allowed or link target not accessible: /var/www/html/
[Wed Jan 31 13:37:12.940769 2018] [core:error] [pid 13798:tid 140181500987136] [client ::1:4867] AH00037: Symbolic link not allowed or link target not accessible: /var/www/html/

system call

Ruby 的 Proces.pid 对应着 system call 的 C 函数 getpid()

父进程

系统中运行的每一个进程都有对应的父进程，每个进程都知道其父进程的标识符(ppid)。在 ruby 中查看父进程，使用 Process.ppid , 对应着system call 中的 getppid() 

puts Process.ppid

多数情况下父进程就是调用它的进程 ，比如你在shell中调用了ls，ls的父进程便是shell。

文件描述符

Unix 世界中，除了将一般的文件视为文件外，也将设备、管道、socket 视为文件(和称为“资源”)。

在Unix世界中，万物皆为文件

在任何进程(程序)中，打开资源都会获得文件描述符。进程关闭后，该进程打开的资源会被一同关闭。

ruby 中通过 fileno 获得文件描述符 

passwd = File.open('/etc/passwd' )
passwd.fileno

输出: 7。进程打开的资源都会获得一个用于标识的唯一数字，内核通过这个数字追踪进程所用的资源。

打开多文件时 

file1 = File.open('/etc/passwd')
file2 = File.open('/etc/hosts')

puts file1.fileno # 7
puts file2.fileno # 8

file1.close
file2.close

file2 = File.open('/etc/hosts')

puts file2.fileno # 7

输出结果为： 

7
8
7
  1. 当我们打开第一个文件时获得描述符编号是7，即尚未使用的最小值。第二个文件描述符是8
  2. 当我们关闭一个资源时，对应的文件描述符编号就关闭了。再次打开 file2 ，文件描述符变为7

这里我们可能会疑惑为什么第一个打开的资源描述符已经是 7 了，而不是 0 或者 1。在 ruby 中可以通过 File.for_fd() 查看标识符对应的资源： 

File.for_fd(3) 
File.for_fd(4)  
File.for_fd(5)  
File.for_fd(6)

运行以上代码出现了错误，3-6 号文件描述符被 Ruby 解释器使用了，并不能被连接 

The given fd is not accessible because RubyVM reserves it

标准流

再看 0-2 号文件描述符, 事实上0-2描述符代表了标准流 STDIN STDOUT STDERR 

puts STDIN.fileno
puts STDOUT.fileno
puts STDERR.fileno

0
1
2

每个Unix进程都有三个必定打开的资源`STDIN` `STDOUT` `STDERR`（标准输入、标准输出、错误）。

  1. STDIN 提供了从键盘，管道中读取输入的通用方法
  2. STDOUT STDERR 提供了一种向显示器、文件、打印机等输出写入内容的通用方法

在STDIN出现之前，为了能够支持键盘，你得在程序中列入一个键盘驱动程序

文件描述符和标准流是使用套接字、管道等进行网络编程的核心所在，也是文件系统操作的核心

进程的资源限制

一个问题：一个进程可以拥有多少个文件描述符？

一个进程拥有的文件描述符是有限的，内核为进程添加了某些资源限制。 以下代码显示了所允许的文件描述符最大连接 

p Process.getrlimit(:NOFILE)

本人计算机的输出结果为： [1024, 65536 ] 第一个为文件描述符的软限制，第二个为文件描述符的硬限制

软限制，硬限制

  1. 软限制的是 可以修改的 ，超出了软限制会抛出异常
  2. 硬限制必须拥有超级用户权限才可以修改

修改软限制 

Process.setrlimit :NOFILE, 2048
p Process.getrlimit(:NOFILE)  #[2048, 2048]

输出: [2048, 2048]

超出限制 

Process.setrlimit(:NOFILE, 3)
File.open( '/dev/null' )

超出限制抛出异常 Too many open files @ rb_sysopen - /dev/null (Errno::EMFILE)

多数程序不需要修改资源限制，但有些情况会需要，比如网络压力测试时同时创建5000个并发连接。

其他资源限制

Process.getrlimit(:NPROC) # 当前用户所允许的最大并发进程数
Process.getrlimit(:FSIZE) # 可以创建的最大的文件
Process.getrlimit(:STACK) # 用于进程栈的最大段的大小
