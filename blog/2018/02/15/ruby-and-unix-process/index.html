Ruby Unix By Example

这篇博文是理解 Unix 进程的笔记，如果你不想阅读二手文章，请看原版书。看完这篇文章或者《理解 Unix 进程》并不能让您深入理解进程，您还是需要阅读操作系统相关书籍。

进程

进程是什么鬼?

进程是 Unix 的核心，代码都是在进程中执行的。当你运行一个ruby程序时，~ruby example.rb~ 你的代码就生成了一个新的进程，程序执行完毕，进程也随之退出。同样，运行 MySQL、PG 等数据库时，打开浏览器时，这些程序都运行在自己的进程中。

pid?

每个进程都有唯一的进程标识符（pid），一个数字。比如一个 ruby 程序中我们可以这样查看它的pid。 

p Process.pid
p $$

$$又是什么鬼

p $$ 等同于 p Process.pid ， $$ 这种用法来源于 perl 和 bash。

有没有法子看看别的程序的pid

我们可以通过 ps 命令查看进程及相关信息，比如 ps -l 显示当前真进程， ps -al 列出其他用户进程信息， 等等其他用法。

pid在真正的开发中有用吗?

实际应用过程中，我们常常会在日志文件中发现pid。比如 apache 服务器的错误日志 /var/log/apache2/error.log 。 

[Wed Jan 31 13:36:52.665744 2018] [core:error] [pid 13639:tid 140581988599552] [client ::1:4866] AH00037:

进程, whosyourdaddy?

系统中运行的每一个进程都有对应的 父进程 ，每个进程都知道其父进程的标识符(ppid)。在 ruby 中查看父进程，使用 Process.ppid 。 

puts Process.ppid

多数情况下父进程就是调用它的进程 ，比如你在shell中输入了ls，ls的父进程便是shell。

文件描述符

Unix 世界中，除了将一般的文件视为文件外，也将设备、管道、socket 视为文件(和称为“资源”)。

在Unix世界中，万物皆为文件

在任何进程(程序)中，打开资源都会获得文件描述符。进程关闭后，该进程打开的资源会被一同关闭。

ruby 中通过 fileno 获得文件描述符 

passwd = File.open('/etc/passwd' )
passwd.fileno

输出: 7。进程打开的资源都会获得一个用于标识的唯一数字，内核通过这个数字追踪进程所用的资源。

打开多文件时 

file1 = File.open('/etc/passwd')
file2 = File.open('/etc/hosts')

puts file1.fileno # 7
puts file2.fileno # 8

file1.close
file2.close

file2 = File.open('/etc/hosts')

puts file2.fileno # 7

输出结果为： 

7
8
7
  1. 当我们打开第一个文件时获得描述符编号是7，即尚未使用的最小值。第二个文件描述符是8
  2. 当我们关闭一个资源时，对应的文件描述符编号就关闭了。再次打开 file2 ，文件描述符变为7

这里我们可能会疑惑为什么第一个打开的资源描述符已经是 7 了，而不是 0 或者 1。在 ruby 中可以通过 File.for_fd() 查看标识符对应的资源： 

File.for_fd(3) 
File.for_fd(4)  
File.for_fd(5)  
File.for_fd(6)

运行以上代码出现了错误，3-6 号文件描述符被 Ruby 解释器使用了，并不能被连接 

The given fd is not accessible because RubyVM reserves it

标准流

再看 0-2 号文件描述符, 事实上0-2描述符代表了标准流 STDIN STDOUT STDERR 

puts STDIN.fileno
puts STDOUT.fileno
puts STDERR.fileno

0
1
2

每个Unix进程都有三个必定打开的资源`STDIN` `STDOUT` `STDERR`（标准输入、标准输出、错误）。

  1. STDIN 提供了从键盘，管道中读取输入的通用方法
  2. STDOUT STDERR 提供了一种向显示器、文件、打印机等输出写入内容的通用方法

在STDIN出现之前，为了能够支持键盘，你得在程序中列入一个键盘驱动程序

文件描述符和标准流是使用套接字、管道等进行网络编程的核心所在，也是文件系统操作的核心

进程的资源限制

一个问题：一个进程可以拥有多少个文件描述符？

一个进程拥有的文件描述符是有限的，内核为进程添加了某些资源限制。 以下代码显示了所允许的文件描述符最大连接 

p Process.getrlimit(:NOFILE)

本人计算机的输出结果为： [1024, 65536 ] 第一个为文件描述符的软限制，第二个为文件描述符的硬限制

软限制，硬限制

  1. 软限制的是 可以修改的 ，超出了软限制会抛出异常
  2. 硬限制必须拥有超级用户权限才可以修改

修改软限制 

Process.setrlimit :NOFILE, 2048
p Process.getrlimit(:NOFILE)  #[2048, 2048]

输出: [2048, 2048]

超出限制 

Process.setrlimit(:NOFILE, 3)
File.open( '/dev/null' )

超出限制抛出异常 Too many open files @ rb_sysopen - /dev/null (Errno::EMFILE)

多数程序不需要修改资源限制，但有些情况会需要，比如网络压力测试时同时创建5000个并发连接。

其他资源限制

Process.getrlimit(:NPROC) # 当前用户所允许的最大并发进程数
Process.getrlimit(:FSIZE) # 可以创建的最大的文件
Process.getrlimit(:STACK) # 用于进程栈的最大段的大小

进程皆有环境

环境指的是环境变量，环境变量是包含进程数据的键值对，所有进程都从父进程处继承环境变量。

在 shell 中设置环境变量传递给 ruby: 

MESSAGE='wing it' ruby -e "puts ENV['MESSAGE']"

同样可以在ruby中设置环境变量被系统调用 

ENV['MESSAGE'] = 'wing it'
system 'echo $MESSAGE'

ENV 是 Hash 吗？ 

p ENV.has_key?('PATH')
p ENV.is_a?(Hash)

true
false

ENV 实现了 Enumerable 和部分 Hash API，但并非全部

进程的参数

Ruby 中，所有进程都可以访问 ARGV 的数组，该数组包含了参数。其他语言可能略有不同，但都会有 argv。 

ARGV.include?('--help')
ARGV.include?('-c')

进程名和退出码

进程名

进程之间没什么办法进行信息交换，除了进程名和退出码 。

进程名可以通过 $PROGRAM_NAME 和 $0 获得，这两个值还可以被改写 

p $PROGRAM_NAME == $0
$0 = "undefined"
p $PROGRAM_NAME

: true
: undefined

退出码

进程结束时会留下退出码(0-255的数字)，用于表明程序是否顺利结束。 退出码0表示顺利进行 。

在 ruby 中，使用 exit 即可退出进程，你可以在 exit 后跟一个数字表示退出码。 

exit 22
  1. at_exit 让进程退出时可以执行一段代码块 
    at_exit {puts 'exit'}
exit
  2. exit! 退出进程，退出码设置为 1，并且不调用 at_exit
  3. abort 同样将退出码视为 1，可以直接输出消息，也可以调用 at_exit 
    abort "exit process"
  4. 出现异常时，并不会立即结束进程，但如果没有代码处理异常，进程就会被终结 
    raise "raise hell"
