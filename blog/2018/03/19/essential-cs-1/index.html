1.3.3 注释

  1. /**注释**/ xml 多行注释，生成单独文件
  2. /// xml 单行注释

编程史上有一段时间认为：代码应该有详细的注释。但时代变了， 可读性好代码比需要测试的代码更有价值 。

规范：

  1. 不要使用注释，除非代码一言难尽
  2. 编写清晰的代码，而不是通过注释澄清复杂的代码

2.1.4 字面量

源代码中的固定值，比如 System.Console.WriteLine(42) ，42就是字面量。

默认情况下输入小数点字面量，被编译器解释为double类型；整数被解释为int类型。

16进制

比如 0x002A

数值格式化为16进制 

class Program {
    static void Main(string[] args) {
	System.Console.Write($"0x{42:X}");
    }
}
//0x2A
