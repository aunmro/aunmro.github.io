Chapter 1 计算机科学与编程

计算过程

计算机科学是围绕计算过程的科学。当计算机忙于做某些事时，一个过程就在它的内部进行计算。计算过程远不止算术计算那么简单，那么什么是计算过程？

举个例子：当你用字典查阅 fiduciary 这个词的意思时

  1. 根据字典的索引确定f的位置
  2. 再根据i确定 fi 的位置
  3. 接下来一个一个词地直接向后查找 fiduciary

以上就是一个过程。

关于计算过程，你可能有很多疑问：

  • 如何描述一个过程
  • 如何证明计算过程达到了特殊的效果
  • 几个产生相同计算结果的过程，如何选择一个最好的
  • 如何构建一台自动执行指定过程的机器

接下来我们将讨论这些问题。

Scheme

描述计算过程需要一门编程语言，这里我们将使用 Scheme 语言。

运行 Scheme 编写的程序需要一个解释器，本文使用的是 Racket 。

最简单的 Scheme 程序是一个数字，输入一个数字，Scheme 将返回这个数字。 

#lang racket
12
;; 12

;; 在 Scheme 表示注释

同样可以输入负数，分数，小数。Scheme 中，小数是不精确的近似值，分数是精确的有理数。 

#lang racket
-7
;; - 7
1/3
;; 1/3
3.1415927
;; 3.1415927

Scheme 预先定义了很多过程，输入过程名过程名即可显示，比如 + sqrt 

#lang racket
sqrt
;; #<procedure:sqrt>
+
;; #<procedure:+>

应用一个过程： 

#lang racket
(sqrt 9) ;;3
(+ 3 6) ;;9

过程是由一个用括号包裹的表达式列表组成。 第一个表达式的值是要应用的过程; 剩下的表达式的值是作为应用过程的参数。 应用程序本身就是表达式，因此它们可以嵌套： 

#lang racket
(sqrt (+ 3 6)) ;;3

计算过程 

(+ 3 6) = 9
(sqrt 9) ;;3

一些其他的计算过程: + - * /

练习1.1

用 Scheme 计算下列值 

#lang racket
(* 3 4)
(* (+ 5 3) (- 5 3))
(/ (+ (* (- 17 14) 5) 6) 7)

复杂的表达式可以通过换行和缩进来阐明其结构 ，比如练习1.1的 (/ (+ (* (- 17 14) 5) 6) 7) ,可以改为： 

#lang racket
(/ (+ (* (-17 14)
	 5)
      6)
   7)

我们还需要学会为过程定义名称： 

#lang racket
(define ark-volume (* (* 300 50) 30))
ark-volume ;; 450000

Scheme 解释器首先计算 (* (* 300 50) 30) 的值，得到 450000, 然后 ark-volume 便成了这个值的名称，你可以运行这个过程。

仅仅为计算结果命名还不够。例如，假设我们要计算几个不同项目的成本，包括5％的销售税，我们可以考虑每件商品的价格，计算销售税，并将税额加到原价中： 

#lang racket
(+ 1.29 (* 5/100 1.29))
;; 1.3545
(+ 2.40 (* 5/100 2.40))
;; 2.52

我们可以定义一个类似 sqrt 的过程，通过产品的价格，计算总成本。Scheme 里，通过 lambda 表达式指定一种计算方法： 

#lang racket
(lambda (x) (+ x (* 5/100 x)))
;; #<procedure>
((lambda (x) (+ x (* 5/100 x))) 1.29)
;; 1.3545
((lambda (x) (+ x (* 5/100 x))) 2.40)
;; 2.52

lambda 表达式的格式为 (lambda (参数) (计算过程)) 。我们可以直接把 lambda 表达式当作 sqrt 那样的过程名直接进行计算,如 ((lambda (x) (+ x (* 5/100 x))) 1.29) 。

每次都写一大段 lambda 很麻烦，我们可以为其命名 

#lang racket
(define f (lambda (x) (* x x)))

(define square
  (lambda (x) (* x x)))

(square 3) ;; 9
(square -10) ;; 100

Concrete Abstractions 写于1998年，现在我们可以直接定义带有参数的过程 

#lang racket
(define (square x) (* x x))
(square 10) ;; 100

练习1.2

  1. 为税收计算过程 (lambda (x) (+ x (* 5/100 x))) 命名
  2. 计算 $1.29 和 $2.40 时的总成本

练习1.3 上文中我们定义了 f 和 square ，但我们把 square 定义为 f 时会发生什么？ 

#lang racket
(define f (lambda (x) (* x x)))
(define square f)
f
;; #<procedure:f>
square
;; #<procedure:f>
(square 7)
;; 49

如练习1.3，我们可以为一个过程定义别名

另外一个例子：定义一个求圆柱体体积的过程，参数为 (radius height) 

#lang racket
(define square (lambda (x) (* x x)))

(define cylinder-volume
  (lambda (radius height)
    (* (* 3.1415927 (square radius))    
       height)))

(cylinder-volume 5 4)
;; 314.15927

(cylinder-volume 5 4) 的计算过程如下 

(cylinder-volume 5 4)
(* (* 3.1415927 (square 5)) 4)
(* (* 3.1415927 (* 5 5)) 4)
(* (* 3.1415927 25) 4)
(* 78.5398175 4)
314.15927

下图以图形表示了计算过程：我们可以看出 Scheme 运行计算过程就是在不停的替换

程序为我们提供了一种对不同值进行相同计算的方法，但有时候，我们希望根据不同条件选择特定的计算。 例如，一个简化的所得税，收入在10,000美元以下的人根本不需要支付任何税款，超过 10000 美元 需要交纳收入的百分之二十。 

#lang racket
(define tax 
  (lambda (income)
     (if (< income 10000)
	 0
	 (* 20/100 income))))

(tax 30000)
;; 6000

上面例子需要注意的有

  1. < ，它会计算参数的大小，返回布尔值 true 或者 false 。类似这样的过程还包括 >, =, <=, >=, even?, odd? 等，它们被称为 谓词 。
  2. if 表达式，=(if () () ())= 第一个参数为判断条件，后面2个参数分别为判断结果 true 和 false 时进行的计算过程

收入为 30000 时，交税金额的计算过程： 

(tax 30000)
(if (< 30000 10000) 0 (* 20/100 30000))
(if #f 0 (* 20/100 30000))
(* 20/100 30000)
6000

(< 30000 10000) 的计算结果为 #f ，即 false，接下来执行第三个表达式。如果计算结果是 #t 则是运行第二个表达式。

Chapter 2 递归和归纳

递归(Recursion)

递归实际上是指在在一个计算过程能调用它自身。听起来很玄乎，我们可以通过例子来理解，最能直观显示展示递归例子是阶乘，比如我们求阶乘 n! 

#lang racket
(define factorial 
  (lambda (n)
    (if (= n 1)
	1
	(* (factorial (- n 1))
	   n))))

(factorial 3) ;; 6
