1 算法分析

1.1 到底什么是算法分析？

算法分析是为了分析程序所使用的计算资源，主要分为2个方面：

  1. 分析程序所占用的空间（内存），不过程序需要解决问题的空间一般由问题本身决定
  2. 分析程序执行所需要的时间

先看一个求和程序，从1加100

#include <stdio.h>

int main() {
    int i, sum = 0, n = 100;
    for(i=1; i<=n; i++) {
        sum = sum + i;
    }

    printf("%d", sum);
    return 0;
}

上面一个程序使用了最笨的方法求和，一个数一个数累加起来。我们再编写一个通过等差数列公式求和的函数来与此函数进行比较：

#include <stdio.h>

int main() {
    int sum = 0, n = 100;
    sum = (1+n)*n/2;
    printf("%d", sum);
    return 0;
}

毫无疑问，程序2优于程序1，但我们得说明为什么程序2优于程序1，因此我们需要一种方法来比较这两个程序―大O表示法。

2 大O表示法和时间复杂度

以程序1为例，我们分析一下这个程序所进行的计算：

#include <stdio.h>

int main() {
    int i, sum = 0, n = 100;  // 执行1次
    for(i=1; i<=n; i++) {     // 执行1次
        sum = sum + i;        // 执行n次
    }
    //整个for 循环执行为 n+1

    printf("%d", sum);        // 执行1次
}

整个程序进行了 n+3 次程序执行，而程序 2 进行了3次程序执行操作。

#include <stdio.h>

int main() {
    int sum = 0, n = 100;
    sum = (1+n)*n/2;
    printf("%d", sum);
    return 0;
}

当我们需要累加到 1000 时，程序1需要执行 1000+3 次操作，而程序2还是只需要执行3次操作，所以程序2优于程序1。

当求和的 n 足够大时，我们可以无视程序1中 n+3 的 3，把程序1的时间复杂度表示为 O(n) 。而对于程序2，执行操作3次，当我们遇到常数时，直接将时间复杂度记为1，即 O(1) 。

归纳一下计算时间复杂度的方法

2.1 如何计算时间复杂度

  1. 用常数1代替所有加法常数
  2. 只保留n的最高阶
  3. 最高接n的系数变为1

另一个例子：一共2次循环， n*n ，时间复杂度为 n^2 。

int i, j, x=0, sum=0, n=100;

for(i=1; i<=n; i++) {
    for(j=1; j<=n; j++) {
        x++;
        sum = sum + x;
    }
}
// for i=> n 次
// for j=> n 次

2.2 常见的时间复杂度

2.2.1 常数阶

时间复杂度 O(1)

int sum = 0;
sum = (1+n)*n/2;

2.2.2 线性阶

时间复杂度 O(n)

int i;
for(i=0; i<n; i++) {
    // 
}

2.2.3 对数阶

int count = 1;
while(count < n) {
    count = count * 2;
}

count 初始值为1，在循环中每次乘2，即 \(2^x\) 次方 。判断条件为直到 \(2^x = n\) , 即 \(x=log_2 n\) ，时间复杂度为 \(O(log n)\) 。

2.2.4 平方阶

时间复杂度 O(n^2)

int i, j;
for(i=0; i<n; i++) {
  for(j=0; j<n; j++) {

  }
}

试着分析下段代码的时间复杂度：

int i,j;
for(i=0; i<n; i++) {
     for(j=i; j<n; j++) {
         //
     }
}

i 层循环中进行 n 次操作； 但在 j 循环中初始条件为 j=i ，即每次执行循环的次数是 n-i 次。总共执行操作为 n+(1)+(n-2)+...+1 即 \(\frac{n(n+1)}{2} = \frac{n^2}{2} + \frac{n}{2}\) ,即 \(O(n^2)\)

2.2.5 常见时间复杂度比较

  阶 称呼
12 O(1) 常数阶
2n+3 O(n) 线性阶
\(3n^2+2n+1\) \(O(n^2)\) 平凡阶
\(5\log_{2}n+20\) \(O(\log{n})\) 对数阶
\(2n+3n\log_2 n+19\) \(O(n\log{n})\) nlogn阶
\(6n^3+2n\) \(O(n^3)\) 立方阶
\(2^n\) \(O(2^n)\) 指数阶

比较这些复杂度 \(O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n)\)

2.3 最坏情况与平均情况

举个例子：一个 n 位的 int 数组，我们需要查找其中一个数字，从数组第一个值开始查找，如果第一个就是我们要找的值，这种情况就是最好情况 \(O(1)\) ；我们在最后一个位置找到 => 最坏情况 \(O(n)\) 。

从概率的角度来将，平均时间 n/2 能找到这个数字，即平均情况，即期望的运行时间。
